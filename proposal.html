Figma Prototype Proposal

I plan on doing UX/UI design for my future career, so it would make the most sense to me to implement coding into my project. I’m aiming to create a web-based application so anyone can use it or test it out! Figma is primarily a front-end development tool to create UI elements, but once I get the layout I want, I will code the back-end using JavaScript or HTML & CSS (likely HTML since I’ve worked with it this quarter).

The main project I propose is a type of inventory management system for my Gundam model kit hobby. The plan is to compile a list of built kits, costs, extra supplies, future purchases, direct links to kits I want, and—if I have time—an optional calendar or alert system for upcoming releases and manual entry of new kits with an input interface.

Steps

1 – Set up the prototype in Figma to get a basic layout for reference during coding.

2 – Start coding the skeleton of the layout to define the structure.

3 – Add images, links, and cost data. This step may take the most time due to sizing challenges (possibly implementing a carousel for images).

4 – Style the web app/site with design polish once the functional layout is in place.

Tools and Methodologies

Figma: For prototyping and plug-ins (to be chosen as needed).

YouTube: For tutorials and quick reference guides during development.

Reddit: For niche problems and code-sharing discussions.

Other Forums: Used via Google when Reddit or YouTube fall short.

Documentation of Project

I will document progress in a separate Google Doc for personal notes and also use Git commit messages to track changes once development begins. Screenshots and Figma file references will be part of the documentation.

Concept

I’ve grown more organized in the past two years, and this project reflects that. It’ll help track my Gundam collection, keep me mindful of costs, and—once polished—I’d love to share it with the Gundam community.

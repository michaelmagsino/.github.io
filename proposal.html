Figma Prototype Proposal

I plan on doing UX/UI design for my future career, so it would make the most sense to me to implement coding to my project. I was thinking of a web-based application so anyone can use it/test it out! Figma is primarily a front-end development tool to create UI elements, but once I get the main layout I want I will then code the back-end with either javascript or html & css (maybe html because that is what I have been coding with throughout this quarter).

The main project I would like to propose is a type of inventory management of my hobby in Gundam model kit building. My plan for this is creating and compiling a list of the Gundams I have built, the costs of each kit, the costs of extra supplies, future kits I want to purchase, direct links to each kit I wanna buy, and if I have time, some sort of calendar element/alert of the release schedules of new Gundams that are up for preorder and also am option to manually add in kits I recently got with some sort of interface to take in information of the kits.

Steps

1 - Set up the Prototype on Figma, this step can be relatively quick because all I intend to do is to get a general layout that I will reference as I code.

2 - After initial prototyping, I will start coding the skeleton of the layout, i.e., the framing of where elements will go.

3 - Once I am happy with the layout, I will then add in the images, links, costs, etc. I am assuming this is where I will take the longest because of the sizing of images/text sizing (I may do a carousel of various images for each kit)

4 - If I have time to, I will then focus on the styling of the web app/site to make it look aesthetically pleasing but again, I shall see depending on how step 3 goes.

I will do the documentation on a separate Google Docs file with dates just for ease of reference, but I will then copy/paste it on the Git commit line with each new version (once I start coding, for the Figma Prototyping, I will use screenshots of my work and also the Figma file itself!).

Concept

I have slowly been becoming more of an organized person throughout the last two years, so I might as well implement that going forward within this project. It will not only keep track of what I have built, but also track the costs of each kit (so it forces me to not buy anymore within my spending limit). It’s also something I would want to share with the Gundam building community in the future once I get it polished!
